# Jason Email Patterns

## Global Defaults

Length:
- One-line replies are common.
- Most normal emails are 2-6 short paragraphs.
- Use bullets for requirements, criteria, dates, or lists of questions.

Openings:
- Familiar/casual: `Hey [Name],`
- Professional/admin: `Hi [Name],`
- Very quick reply: no greeting.
- Lowercase `hey` / `hi` is acceptable in casual or transactional messages.

Closings:
- `Thanks,\nJason` for normal admin/professional.
- `Jason` for short or slightly firmer messages.
- `thanks\njason` for quick lowercase transactional messages.
- No closing for one-line replies.

Voice:
- Direct over polished.
- Concrete over diplomatic.
- Light warmth, not performative warmth.
- Honest constraints over vague hedging.

Avoid:
- `I hope you're well`
- `Hope this finds you well`
- `Just wanted to reach out`
- `Touching base`
- `Circling back`
- Excessive apology
- Over-explaining background the recipient does not need

## Common Templates

### Simple Confirmation

```text
This works!
```

```text
Sounds great — I'm in for [event] at [time]. Looking forward to it.

Thanks,
Jason
```

### Schedule Conflict

```text
Hi [Name],

I have a schedule change and won't be able to make [time/event].

Thanks,
Jason
```

### Slight Delay

```text
There's some traffic / Uber delays. I might be about [X] minutes late.
```

### Admin Question

```text
Hi [Name],

Quick question — could you let me know [specific process/info]? I'm [relevant identifier/context].

Thanks,
Jason
```

### Service Issue

```text
Hi [Name],

I don't think [thing] is working. When I [action], [observable behavior]. [Additional concrete evidence.]

Could you take a look or let me know what the next step is?

Thanks,
Jason
```

### Follow-Up

```text
Hi [Name],

Just following up on my email from [date] about [topic]. It's been about [time] and I haven't heard back on [specific blocker].

Can you let me know the next steps?

Jason
```

### Firm Receipt Request

```text
Please confirm receipt of [document/form/items] I sent on [date].

[Specific constraint/problem.]

Let me know how I can proceed.
```

### Professional Decline / Bandwidth

```text
Hi [Name],

Thanks for your patience on this. I've been [brief context].

I need to focus on [priority] for now and won't be able to move forward with [opportunity] at this time. I appreciate the interest — hopefully we can revisit this down the line when my bandwidth opens up.

Jason
```

### Need Internal Clearance

```text
Hi [Name],

Quick question about [event/topic] — do you have [materials/questions/details]? I need to get them cleared by OpenAI's communications team beforehand.

If you have anything from [person/team] or the planned topics, that would be helpful.

Thanks,
Jason
```

### Warm Check-In

```text
Hey [Name],

Just checking in — how's [thing] going?

Jason
```

### Dinner Invite

```text
Hi team,

Would love to do dinner when you're free. Send me 2-3 days that work and I'll make it happen.

Jason
```

### Intro

```text
hey [A] and [B],

introducing you two - [A] [one-line context], [B] [one-line context].

[Why this might be useful / why they should connect.]

[Optional question.]

jason
```

### Founder / Investment Follow-Up

```text
Let me follow up [day].
```

```text
Hey [Name],

Next week works - I'll be in [city]. [Day/time]?

Jason
```

### Assistant Delegation

```text
hey [Name],

[Person/situation] needs [outcome]. i'm at [location/timing]. can you [specific action]? [phone/email/details]. if they say yes, [next action]. this is time sensitive.

thanks
```

### Opt-Out

```text
Stop
```

## Persona Selection Hints

- If the user asks for "nice but short", use Practical Admin or Professional Constraint.
- If the user asks for "make it sound like me", default to shorter and less polished than a generic assistant draft.
- If the user asks for "firm", use Vendor Escalation but stay factual.
- If the recipient is a friend or warm peer, use Warm Network.
- If the recipient is an assistant, use Assistant Delegation with complete operational details.
- If the recipient is a founder, investor, or platform handling a deal, use Investor / Founder Coordination.
- If the user supplies a long messy context dump, compress it aggressively into the one blocker and one ask.
